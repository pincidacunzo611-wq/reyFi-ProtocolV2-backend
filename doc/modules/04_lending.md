# Lending 服务 — 借贷协议

## 1. 概述

Lending 服务处理 **去中心化借贷** 相关的数据。用户可以：
- **存款**：将代币存入借贷池赚取利息
- **借款**：以存款为抵押借出另一种代币
- **还款**：偿还借款
- **被清算**：当抵押率不足时，仓位会被清算

> **类比理解**：银行的存贷款系统。你存钱进去有利息（供应 APR），你可以抵押资产借钱（借款 APR > 供应 APR，银行赚差价），如果你的抵押物贬值太多（健康因子 < 1），银行会强制卖掉你的抵押物来还债（清算）。

---

## 2. 核心概念

| 概念 | 解释 | 例子 |
|------|------|------|
| **供应 APR** | 存款年化收益率 | 存 ETH 年化 3.5% |
| **借款 APR** | 借款年化利率 | 借 USDC 年化 5.2% |
| **利用率** | 已借出金额 / 总存款 | 80% = 高利用率 |
| **抵押因子** | 能借出多少比例 | 0.75 = 存100能借75 |
| **健康因子** | 抵押物价值 / 负债价值 × 清算阈值 | < 1.0 可被清算 |
| **清算罚金** | 被清算时的额外成本 | 5% = 多付5%给清算人 |

---

## 3. 事件消费

| 链上事件 | 消费逻辑 |
|---------|---------|
| **Deposit** | 更新 `lending_user_positions.supplied_amount` |
| **Withdraw** | 减少用户存款额 |
| **Borrow** | 更新 `lending_user_positions.borrowed_amount` |
| **Repay** | 减少用户借款额 |
| **LiquidationCall** | 写入 `lending_liquidations` 表 + 更新用户仓位 |

每个事件处理后都会：
1. 更新 `user_activity_stream`（用户活动流）
2. 清除相关 Redis 缓存

---

## 4. 对外接口

| RPC 方法 | 功能 |
|----------|------|
| `GetMarkets` | 所有借贷市场列表（含 APR、TVL、利用率） |
| `GetMarketDetail` | 单个市场详情 + 利率历史曲线 |
| `GetUserPosition` | 用户的存/借仓位 + 健康因子 |
| `GetLiquidations` | 清算记录列表 |
| `BuildSupply/Borrow/Repay/Withdraw` | 构造链上交易数据 |

---

## 5. 数据库表

| 表 | 说明 |
|----|------|
| `lending_markets` | 借贷市场配置（利率模型参数、清算阈值等） |
| `lending_market_snapshots` | 市场数据快照（总存/借、APR、TVL） |
| `lending_user_positions` | 用户仓位（存了多少、借了多少、健康因子） |
| `lending_liquidations` | 清算记录（谁被清算、清算人、金额） |

## 6. 健康因子计算

```
健康因子 = Σ(存款金额 × 价格 × 清算阈值) / Σ(借款金额 × 价格)

例如：
  存款: 10 ETH × $3000 × 0.85(阈值) = $25,500
  借款: 15,000 USDC × $1 = $15,000
  健康因子 = 25,500 / 15,000 = 1.70 ✅ 安全

  如果 ETH 跌到 $1,800:
  健康因子 = 10 × 1800 × 0.85 / 15000 = 1.02 ⚠️ 危险

  如果 ETH 跌到 $1,750:
  健康因子 = 10 × 1750 × 0.85 / 15000 = 0.99 ❌ 可被清算
```
